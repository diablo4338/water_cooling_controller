# Architecture

## Layers
- `ble_app/core.py` — публичный BLE API и бизнес‑логика протокола. Этот слой считается стабильным: GUI и тесты используют только его.
- `ble_app/presentation.py` — слой состояния UI (Actions, AppState, derive_ui). Чистая логика без Qt.
- `ble_app/gui.py` — UI (PySide6) и `BleWorker`, который вызывает `BleAppCore`. GUI работает через `AppModel` и Actions.
- `ble_app/main.py` — минимальный entrypoint, только запускает GUI.

## Rationale
BLE библиотеки ведут себя нестабильно, поэтому тесты и GUI должны использовать один и тот же слой логики. Это снижает расхождения между “как работает приложение” и “как написаны тесты”.
UI‑логика вынесена в `presentation.py`, чтобы тестировать состояние и доступные действия без зависимости от PySide6.

## Public API Contract
Любые изменения в `ble_app/core.py` должны сопровождаться корректировкой тестов. Этот файл является источником истины для BLE‑поведения приложения.
Actions из `ble_app/presentation.py` — публичный контракт между GUI и UI‑логикой. Переименование Actions требует обновления биндинга UI и тестов.
Все интерактивные элементы GUI помечаются `actionId = Action.name` и используются для smoke‑проверок.
