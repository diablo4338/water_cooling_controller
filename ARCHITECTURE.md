# Architecture

## Layers
- `ble_app/core.py` — публичный BLE API и бизнес‑логика протокола. Этот слой считается стабильным: GUI и тесты используют только его.
- `ble_app/gui.py` — UI (PySide6) и `BleWorker`, который вызывает `BleAppCore`.
- `ble_app/main.py` — минимальный entrypoint, только запускает GUI.

## Rationale
BLE библиотеки ведут себя нестабильно, поэтому тесты и GUI должны использовать один и тот же слой логики. Это снижает расхождения между “как работает приложение” и “как написаны тесты”.

## Public API Contract
Любые изменения в `ble_app/core.py` должны сопровождаться корректировкой тестов. Этот файл является источником истины для BLE‑поведения приложения.
